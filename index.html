Glosario de comandos Git

Comando Descripción Parametros Ejemplo
GIT Y REPOSITORIOS
git --version Ver la versión de Git y comprobar si está instalado. Ninguno git --version
git config --global user.name [nombre] Establecer un nombre que sea identificable para cuando se revise el historial de versiones. --global, indica que se configura una opción de forma global, user.name, es la clave que se usa para establecer un nombre que identifica al usuario asociado a Git. git config --global user.name "John Smith"
git config --global user.email [email] Establecer una dirección de correo electrónico que se asociará a cada commit del historial. --global, indica que se configura una opción de forma global, user.email, es la clave que se usa para establecer la dirección de correo asociada a Git. git config --global user.email jogicarrilloru@ittepic.edu.mx
git init Inicializar un directorio existente como repositorio Git. Ninguno git init
git clone [url-repository] Recuperar un repositorio completo desde una ubicación alojada a través de una URL. [url] Enlace del repositorio remoto que será clonado. git clone https://github.com/josecar2505/iw2023ittepic.git
git clone [url] . Al añadir un punto al final, el clonado se hace en la carpeta donde lanzamos la orden sin crear una subcarpeta. [url] Enlace del repositorio remoto que será clonado. git clone https://github.com/josecar2505/iw2023ittepic.git .
ESTADOS Y CONSULTAS
git status Este comando te informa del estado de los archivos de tu proyecto y listará los archivos en verde o en rojo. Los archivos en verde se han añadido a la preparación pero no se han confirmado todavía. Los archivos marcados en rojo son aquellos que no han sido añadidos a la preparación. Ninguno. git status
git log Muestra todos los commits de la rama actual Ninguno git log
git log --oneline Muestra una lista de las confirmaciones (commits) en el historial del repositorio. Cada confirmación se presenta en una sola línea, lo que facilita la lectura y el seguimiento de las últimas confirmaciones. --oneline. Indica que se desea ver la salida en una sola línea por confirmación. git log --oneline
git checkout [identificador-commit] Ver un commit mediante su identificador [identificador-commit] Es el identificador de la confirmación a la que deseas moverte. git checkout def5678
git reflog Lista ordenada de los commits realizados. Este historial nos sirve para deshacer cambios en el repositorio. Ninguno. git reflog
git checkout -- Volver al estado original. Reemplaza los cambios en tu directorio de trabajo con el último contenido de HEAD. Ninguno. git checkout --
git commit --amend -m “message” Se utiliza en Git para realizar una modificación en la confirmación más reciente (la última confirmación) en tu historial. --amend: Indica que deseas realizar una modificación a la última confirmación. -m "message": Permite especificar el nuevo mensaje que deseas asociar a la última confirmación. git commit -m "Confirmación inicial con mensaje incorrecto"
git restore --staged [file-name] Se utiliza para deshacer cambios de un archivo del área de preparación (staging area) y volverlos al estado sin preparar (unstaged) en tu directorio de trabajo --staged: Indica que deseas deshacer los cambios que están en el área de preparación. file-name: Es el nombre del archivo que deseas quitar del área de preparación. git restore --staged mi-archivo.txt
COMMITS
git add [file-name] Agregar el archivo seleccionado al stage. [file-name]: el nombre del archivo que deseas agregar al área de preparación. git add archivo1.txt
git add . Agregar todos los archivos modificados al stage. Ninguno. git add .
git reset Deshacer los cambios locales en el estado de un repositorio de Git. Ninguno. git reset
git reset [file-name] Descartar un archivo del stage. [file-name]: indica el nombre del archivo que será descartado. git reset archivo.txt
git commit -m "[description]" Crear un commit de los archivos en stage, con una descripción breve "[description]" git commit -m "Agregado archivo1.txt y archivo2.txt"
git commit -am "[description]" Crear un commit de los archivos en seguimiento, con una descripción breve. "[description]" git commit -am "Agregado un nuevo archivo y actualizado archivo existente"
git push origin [branch] Subir al repositorio los cambios de la rama [branch] [branch]: identifica el nombre de la rama. git push origin main
git push --force Ignorar los cambios locales y hacer push Ninguno. git push --force
git push --all Subir al repositorio todas las ramas --all indica que deseas empujar todas las ramas locales al control remoto. git push --all
git fetch Traer los commits del repositorio remoto al repositorio local Ninguno. git fetch
RAMAS
git branch Crear una rama desde la rama actual [branch-name]: Identifica el nombre de la nueva rama que será creada. git branch nuevas-caracteristicas
git branch [branch-name] [initial-branch] Crear rama a partir de otra rama [new-branch-name] es el nombre de la nueva rama que deseas crear. [starting-branch] es la rama desde la cual deseas crear la nueva rama. git branch caracteristica-nueva desarrollo
git branch -m [branch-name] [new-branch-name] Cambiar de nombre una rama -m especifica que la rama [branch-name] cambiará de nombre a [new-branch-name] git branch -m experimento nueva-experiencia
git branch -d [branch-name] Eliminar una rama [branch-name] identifica el nombre de la rama que se quiere eliminar y -d especifica que se trata de una eliminación. git branch -D feature-branch
git checkout [branch-name] Moverse a una rama concreta [branch-name]: identifica el nombre a la rama a la que se quiere mover. git checkout feature-branch
git checkout -b [branch-name] Crear una rama y moverse a esa rama -b se usa para crear una nueva rama con el nombre de [branch-name] y cambiar a esa nueva rama. git checkout -b nueva-caracteristica
git merge [branch-name] Unir la rama actual con la rama seleccionada [branch-name] representa el nombre de la rama que deseas fusionar en la rama actual. git merge feature-branch
git branch -av Visualizar las ramas Ninguno. git branch -av
ETIQUETAS
git tag -a [tag-name] -m “Message” Crear una etiqueta -a: indica que estás creando una etiqueta anotada. -m "Message": permite especificar un mensaje descriptivo que se asociará con la etiqueta. git tag -a v1.0 -m "Versión 1.0: Lanzamiento inicial del proyecto"
git tag Mostrar las etiquetas Ninguno. git tag
git show [tag-name] Mostrar características de una etiqueta en concreto [tag-name]: nombre de la etiqueta que deseas mostrar. git show v1.0
git push --tags Subir las etiquetas al repositorio remoto Ninguno. git push --tags

Conclusión

Los comandos en Git son una parte fundamental para la gestión exitosa de proyectos de desarrollo de software. Su importancia radica en que proporcionan un control total sobre la versión de tu código, permiten colaborar eficazmente con otros miembros del equipo, previenen errores costosos y mejoran la productividad en el flujo de trabajo de desarrollo. Conocer y dominar estos comandos no solo es valioso, sino esencial para cualquier desarrollador o equipo que utilice Git como sistema de control de versiones.
